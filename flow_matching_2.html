Part 2 — The Three Mathematical Objects
// Probability density paths · Vector fields · Flows — and how they connect
The Three Core Objects

Flow matching is built on three tightly connected mathematical objects. You need all three to understand what the model is doing. Each one tells you something different about the transformation from P₀ to P₁.

p
Object 1 — Probability Density Path
p_t : ℝd → ℝ+ where t ∈ [0, 1]
A time-dependent density function. For any point x in data space and any time t, it tells you: what is the probability density at this location at this moment in time? It traces a continuous path of distributions from P₀ (Gaussian noise) at t=0 to P₁ (data) at t=1.
📍 Boundary conditions: p at t=0 must equal P₀ (our Gaussian source). p at t=1 must equal P₁ (the data distribution, approximated by a Gaussian mixture).
v
Object 2 — Time-Dependent Vector Field
v_t : ℝd → ℝd where t ∈ [0, 1]
At any location x and time t, the vector field says: in which direction, and how fast, should a particle at position x move right now? Think of it as a map of wind speeds and directions — every point in space has an arrow attached showing where to go.
🧭 The vector field is time-dependent — the arrows change as t progresses. A particle must follow a changing map. However, with the optimal transport choice (Part 3), the direction turns out to be constant over time!
φ
Object 3 — Flow (Diffeomorphic Map)
φ_t : ℝd → ℝd where t ∈ [0, 1]
The flow is the actual path each point takes as it moves through space from t=0 to t=1. It's generated by integrating the vector field over time — if the vector field is the wind, the flow is the trajectory the leaf takes. It's a smooth, invertible map (diffeomorphism).
🌊 Defining equations: φ₀(x) = x (every point starts at itself). Then d/dt φ_t(x) = v_t(φ_t(x)) — the rate of change of position equals the vector field at the current position.
How the Three Objects Connect
The Defining Equation
dφ_t(x) ─────── = v_t(φ_t(x)) dt φ₀(x) = x ← every point starts at itself at t=0

This is essentially an ODE. The vector field v_t drives the flow φ_t. To follow the flow for a single starting point x:

  1. You're at position x at time t
  2. Look up the vector field: v_t(x) tells you direction and speed
  3. Take a tiny step in that direction: new position ≈ x + ε·v_t(x)
  4. Now t has advanced by ε. Repeat — but the vector field may have changed!
  5. Continue until t = 1. You've arrived at your sample from P₁.
⚙️ In practice you don't take tiny steps manually — you use an ODE solver (like Runge-Kutta) which does this automatically and efficiently. Straighter paths = fewer solver steps needed.

At time t: each point has an arrow saying "go here"

How the density moves

If you take every point in P₀ and push it through the flow, you get p_t. Mathematically: push-forward of the original density through φ_t equals p_t.

p_t = (φ_t)₊ p₀ ← "push forward"

Move all the mass of P₀ through the flow → you trace out the entire probability path p_t.

Conditional vs Marginal — The Key Simplification
🔑 The central trick of the paper: Instead of defining paths for the entire distributions P₀ → P₁ at once, define paths for individual pairs of samples (one from P₀, one from P₁). Then aggregate. This makes the whole thing tractable.
Conditional probability path
p_t(x | x₁) ← conditioned on ONE data point x₁

Pick one data sample x₁. Now define a path from the source Gaussian to a small Gaussian centered on that single data point. Simple!

Conditional path for one data point x₁
Boundary conditions
t=0:  p₀(x|x₁) = 𝒩(0, I) — same as source, ignoring x₁
t=1:  p₁(x|x₁) = 𝒩(x₁, σ²) — tight gaussian around data point
Marginal probability path
p_t(x) = ∫ p_t(x|x₁) · q(x₁) dx₁

The full distribution path is just the average over all data points. For each data sample x₁ in your dataset, compute the conditional path, then average them all (weighted by the data distribution q(x₁)).

💡 In practice: sample one x₁ from your dataset per training step. You don't need to do the full integral — just sampling works because of the law of large numbers.
Marginal vector field
u_t(x) = ∫ u_t(x|x₁) · p_t(x|x₁)/p_t(x) · q(x₁) dx₁

Similarly, the full vector field is an average of the conditional vector fields, reweighted by how likely each x₁ is given the current position x at time t.

🔑 Key theorem: The marginal vector field generates the marginal probability path. Prove the conditional case works → the aggregate automatically works.
Aggregating conditional paths → full distribution flow

SUMMARY OF PART 2

Three objects: p_t (density path), v_t (vector field), φ_t (flow). The vector field drives the flow. The flow transports the density. Define these for single pairs of samples (conditional), then aggregate to get the full distribution flow.
