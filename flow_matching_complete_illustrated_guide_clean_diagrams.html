1The big picture

A generative model tries to learn the distribution of real data. If your dataset contains dogs wearing hats, the model wants to learn the shape of all valid dog-with-hat images. In a toy 2D world, the data distribution looks like blobs or clusters. In a real high-dimensional world, those blobs live in a huge vector space of pixels or latents.

The central problem is: how do we turn something easy to sample, like Gaussian noise, into something that looks like real data?

The model wants to learn how to sample from these structured regions rather than random junk.

2How diffusion solves generation

Forward process

real sample
   -> add a little noise
   -> add more noise
   -> add more noise
   -> almost pure Gaussian noise

Reverse process

Gaussian noise
   -> denoise a bit
   -> denoise more
   -> denoise more
   -> final sample
Diffusion works very well, but it commits to one particular corruption geometry.
Key idea: diffusion chooses a fixed corruption path and learns to reverse it.

3What flow matching changes

Flow matching says: maybe we do not need to define a specific noising process first. Instead, we can directly learn a rule that tells a sample how to move through space over time. That rule is a velocity field.

So instead of “learn to reverse noise,” the idea becomes “learn how to transport a simple source distribution into the data distribution.”

Flow matching gives you freedom to choose better path geometry than the one inherited from a noising process.

4Particle intuition

5What is a vector field?

A vector field is a function that assigns an arrow to every point in space. The arrow says which direction to move and how fast to move. If you keep following those arrows, you trace out a trajectory. That trajectory is the flow.

6The 3 core objects

Probability density path

p_t(x) tells you how much probability mass is at location x at time t.

Vector field

v_t(x) tells you how a point should move right now.

Flow

φ_t(x) is the actual trajectory obtained by following the vector field.

Often indirect trajectoriesCan use simpler trajectories

8School hallway analogy

Imagine students start randomly in a school lobby and need to end up in classrooms.

Diffusion mindset: first scramble them in a specific way, then learn how to unscramble.

Flow matching mindset: directly learn the rule “if a student is here at time t, walk this way.”

9We do not know the exact data density

We do not usually know a closed-form formula for the data distribution. We only have samples from it. That means we cannot easily write down the exact global flow mathematically. We need a training trick that only relies on samples.

10The key trick: single source-target pairs

Instead of defining a flow between whole distributions at once, define a path between a source sample x_0 from Gaussian noise and a target sample x_1 from real data. This pair-specific path is called a conditional path.

The paper proves that learning from these conditional paths is enough to recover the right global objective.

11Straight-line conditional paths

The cleanest conditional path is linear interpolation:

x_t = (1-t)x_0 + t x_1

At t=0, you are at x_0. At t=1, you are at x_1.

12Velocity on a straight line

Differentiate the straight-line path with respect to time:

u_t = d x_t / dt = x_1 - x_0

That means the velocity is constant along the whole path. The sample keeps moving in the same direction all the way from source to target.

13Why conditionals solve tractability

The global flow objective over whole distributions is hard. But training on pairwise paths is easy:

  1. sample x_0
  2. sample x_1
  3. sample t
  4. compute x_t
  5. compute the target velocity

The paper proves that optimizing the conditional objective reaches the same optimum as the intractable marginal objective.

14Why the conditional objective works

Over many training pairs, the model sees many local directions. At each location and time, it learns the correct averaged local movement induced by the whole dataset. That aggregated local direction is the global vector field.

15The training loss

  1. Sample x_0 ~ N(0, I)
  2. Sample x_1 ~ p_data
  3. Sample t ~ Uniform(0,1)
  4. Compute x_t = (1-t)x_0 + tx_1
  5. Set target velocity u_t = x_1 - x_0
  6. Train vθ(x_t, t) to match u_t

L = E[ || vθ(x_t,t) - (x_1-x_0) ||^2 ]

16Why the loss makes sense

At point x_t on the path from x_0 to x_1, the correct answer to “which direction should I move?” is “keep moving toward the target along the path.” The loss just teaches the network that local rule.

17Tiny numeric example

x0 = [0.0, 0.0]
x1 = [10.0, 4.0]
t  = 0.3

xt = (1 - 0.3) * x0 + 0.3 * x1
   = [3.0, 1.2]

ut = x1 - x0
   = [10.0, 4.0]

18PyTorch snippet: one training example

import torch

x0 = torch.tensor([0.0, 0.0])
x1 = torch.tensor([10.0, 4.0])
t = torch.tensor(0.3)

xt = (1 - t) * x0 + t * x1
ut = x1 - x0

19What the neural network learns

The neural network learns a function vθ(x,t): input current position and time, output the local velocity vector.

20Sampling after training

Once trained, generation works like this:

  1. sample x(0) ~ N(0, I)
  2. query the model for local velocity
  3. integrate the ODE until t=1

21Why ODE language appears

The model defines dx/dt = vθ(x,t). That is an ordinary differential equation. The ODE solver follows the trajectory implied by the field.

22Euler integration intuition

x = x + dt * v_theta(x, t)

23PyTorch pseudocode for sampling

x = sample_gaussian(batch_size, dim)

num_steps = 100
dt = 1.0 / num_steps

for step in range(num_steps):
    t = step / num_steps
    v = model(x, t)
    x = x + dt * v

24Why flow matching can be easier than diffusion

  • trajectories can be simpler
  • vector fields can be smoother
  • sampling may need fewer function evaluations
  • training targets can be more direct

25The path is chosen by us

Flow matching is a framework. It does not force one path. You pick a path family: diffusion-like Gaussian paths, straight-line paths, or other custom interpolations.

26Diffusion as a special case

If you choose certain time-dependent Gaussian paths, the flow matching objective becomes equivalent to diffusion-style objectives. That is why people say flow matching generalizes diffusion.

27Conditional Gaussian paths

More generally, a conditional path can be a whole Gaussian distribution at each time, not just a point on a line: p_t(x | x_1) = N(x | μ_t(x_1), σ_t^2 I).

28Marginalizing over conditionals

“Marginalizing over conditionals” means averaging over all possible target data samples. Each target sample defines one conditional path. Put them together and you recover the global probability path.

29Why the conditional objective is practical

The full marginal flow objective requires hard integrals over distributions. The conditional objective only needs sampled pairs, random times, and local velocities.

30Cleanest mental model

1. Pick a real data point x1
2. Pick a random noise point x0
3. Choose a path from x0 to x1
4. Pick a random point xt on that path
5. Teach the model the correct velocity at xt
6. Repeat over many pairs

31Toy data batch creation

import torch

def sample_data(batch_size):
    centers = torch.tensor([[-2.0, 0.0],[2.0, 0.0]])
    cluster_ids = torch.randint(0, 2, (batch_size,))
    chosen_centers = centers[cluster_ids]
    noise = 0.3 * torch.randn(batch_size, 2)
    return chosen_centers + noise

def sample_source(batch_size):
    return torch.randn(batch_size, 2)

def make_flow_matching_batch(batch_size):
    x0 = sample_source(batch_size)
    x1 = sample_data(batch_size)
    t = torch.rand(batch_size, 1)
    xt = (1 - t) * x0 + t * x1
    target_velocity = x1 - x0
    return xt, t, target_velocity

32Build a simple PyTorch model

import torch.nn as nn

class FlowMLP(nn.Module):
    def __init__(self, data_dim=2, hidden_dim=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(data_dim + 1, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, data_dim),
        )

    def forward(self, x, t):
        inp = torch.cat([x, t], dim=1)
        return self.net(inp)

33Loss function in code

def flow_matching_loss(model, batch_size):
    xt, t, target_velocity = make_flow_matching_batch(batch_size)
    pred_velocity = model(xt, t)
    loss = ((pred_velocity - target_velocity) ** 2).mean()
    return loss

34Full training loop

import torch.optim as optim

model = FlowMLP()
optimizer = optim.Adam(model.parameters(), lr=1e-3)

num_steps = 5000
batch_size = 256

for step in range(num_steps):
    optimizer.zero_grad()
    loss = flow_matching_loss(model, batch_size)
    loss.backward()
    optimizer.step()

35How to sample after training

@torch.no_grad()
def sample_from_model(model, num_samples=1000, num_steps=100):
    x = torch.randn(num_samples, 2)
    dt = 1.0 / num_steps
    for step in range(num_steps):
        t_value = step / num_steps
        t = torch.full((num_samples, 1), t_value)
        v = model(x, t)
        x = x + dt * v
    return x

37Translate the code into English

At training time: pick a real data point, pick a random noise point, draw a path between them, choose a random point on that path, and teach the model the correct local direction there. At inference time: start from noise, repeatedly ask the model where to move, and follow those directions until you get a sample.

38Why give time t to the network?

Even when straight-line conditional velocity is constant, the full aggregated field can still depend on time. More general path families definitely depend on time. So giving t to the model adds flexibility.

39Relation to CNFs

Continuous normalizing flows (CNFs) are continuous-time generative models defined by ODEs. Flow matching is an effective way to train them.

40Why optimal transport shows up

Optimal transport asks for an efficient way to move one distribution into another. Straight-line or transport-like paths are often more efficient than diffusion-like paths that wander because they inherit the geometry of a noise process.

41Why diffusion trajectories curve

Diffusion first spreads data outward with noise, then shapes that mess back into data during reverse sampling. That often induces curved or indirect paths. Flow matching can skip that and directly choose cleaner transport geometry.

42The key formula to remember

x_t = (1-t)x_0 + tx_1

u_t = x_1 - x_0

L = E[ || vθ(x_t,t) - (x_1-x_0) ||^2 ]

43What the model knows during training

The target velocity uses x_0 and x_1, but the network generally only gets x_t and t. That forces it to learn the average local motion implied by the dataset.

44Why this is not memorizing lines

Different source-target pairs can pass through nearby regions in space. The model cannot memorize every pair separately. Instead, it learns the average field that best explains all those local training targets.

45Geometric aggregation picture

46Why random time sampling works

You do not need to simulate every time along every path during training. Sample a random t. Over many batches, you cover the whole time interval.

47Relation to score matching

Diffusion models are often trained to predict noise or a score function like ∇ log p_t(x). Flow matching instead directly predicts transport velocity.

48Why Stable Diffusion 3 moved this way

Once diffusion objectives were understood inside a broader flow framework, it became natural to ask whether better path choices exist. Straight or transport-like paths can be easier to integrate and more efficient.

49Common beginner confusions

“Are we moving one sample or the whole distribution?”

Both views are valid.

“Do we need the exact density formula?”

No. Samples are enough.

“Is this just interpolation?”

No. Interpolation gives training targets; the learned model becomes a global field.

“Is flow matching identical to diffusion?”

No. Diffusion is one special case inside a larger framework.

50Practical recipe

1. Choose a source distribution, usually Gaussian
2. Choose a path family from source to target
3. Sample x0, x1, and t
4. Compute xt along the path
5. Compute the target velocity ut
6. Train v_theta(xt, t, conditioning) to match ut
7. At inference, integrate dx/dt = v_theta(x, t, conditioning)

51Conditional flow matching

For text-to-image, TTS, class-conditional generation, and other conditional tasks, the network becomes vθ(x, t, cond).

52Latent image generation

Modern systems often work in latent space rather than raw pixel space: image -> encoder -> latent z. Then flow matching is performed on z.

53TTS / audio connection

The same machinery works for audio. The target x_1 might be a mel spectrogram, a speech latent, or embedded acoustic codec vectors. The model learns to transport noise latents into speech latents.

56More realistic conditioned PyTorch skeleton

class ConditionalFlowModel(nn.Module):
    def __init__(self, x_dim, cond_dim, hidden_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(x_dim + 1 + cond_dim, hidden_dim),
            nn.SiLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.SiLU(),
            nn.Linear(hidden_dim, x_dim),
        )

    def forward(self, x, t, cond):
        inp = torch.cat([x, t, cond], dim=-1)
        return self.net(inp)

57Math cheat sheet

p_t(x) = probability density at time t

v_t(x) = local velocity field

φ_t(x) = flow / actual trajectory

x_t = (1-t)x_0 + tx_1 = straight-line path

u_t = x_1 - x_0 = target velocity

L = E[ || vθ(x_t,t) - u_t ||^2 ] = training objective

dx/dt = vθ(x,t) = generation ODE

58Deepest intuition

Flow matching turns generative modeling into learning a motion rule. Instead of saying “learn to undo noise,” it says “learn how a point should move through space over time so that random noise becomes real data.”

59One-sentence explanation

Flow matching trains a neural network to predict the velocity that transports samples from a simple source distribution to the data distribution along chosen paths.
