🤔 What Problem Does This Paper Solve?

Imagine you have a super-smart robot assistant. You want to tell it: "Bring me the rice chips from the drawer."

A normal language model (like the ones behind chatbots) is brilliant at understanding that sentence. But it's blind — it has never seen your kitchen, your drawer, or what rice chips look like.

The Core Problem: Grounding

Grounding means connecting words to the real world. "Red block" means nothing to a robot unless it can SEE what block is red and WHERE it is. Language models trained only on text have no grounding — they don't know what the world looks like right now.

What was tried before PaLM-E?

Approach 1: Text-only LLM + Separate Vision Model

Run a vision model separately, convert what it sees into text ("I see a red block on the left"), then feed that text into the LLM. Problem: You lose precise geometric info — "on the left" is vague; the robot needs exact positions.

Approach 2: Affordance Functions (SayCan)

LLM suggests steps, and a separate learned "affordance" function checks if each step is physically possible. Problem: Requires extra models per skill, doesn't work well for long multi-step plans.

PaLM-E's Answer

Inject raw sensor data (images, robot state vectors, 3D scene representations) directly into the language model's token stream. The LLM processes images and text as one unified sequence. No separate grounding step needed.

🔤 Concept 1: Tokens and Embeddings

Before we can understand PaLM-E, we need to understand how language models turn words into numbers.

Step 1 — Tokenization: Words → Integer IDs

A tokenizer splits text into small chunks called tokens (roughly syllables or short words) and assigns each an integer ID. PaLM uses a vocabulary of 256,000 tokens.

PYTHON
# Think of a tokenizer like this simplified version:
# Real tokenizers use algorithms like BPE (Byte Pair Encoding)

vocab = {
    "bring": 0, "me": 1, "the": 2, "rice": 3,
    "chips": 4, "from": 5, "drawer": 6
}

sentence = "bring me the rice chips"
token_ids = [vocab[w] for w in sentence.split()]
print(token_ids)  # [0, 1, 2, 3, 4]

Step 2 — Embedding: Integer IDs → Dense Vectors

Each token ID is mapped to a vector of numbers (the "embedding"). This vector encodes the meaning of the word in a continuous space. Words with similar meanings end up with similar vectors.

Why vectors?

Neural networks work with numbers, not words. A vector like [0.2, -0.8, 1.3, ...] lets the model do math on meaning — for example, "king" - "man" + "woman" ≈ "queen" in embedding space. This is impossible with plain integer IDs.

PYTORCH
import torch
import torch.nn as nn

# An embedding table: maps each token ID to a learnable vector
vocab_size   = 256_000   # PaLM uses 256k tokens
embed_dim    = 4096      # each token becomes a 4096-dim vector (for 8B model)

# nn.Embedding is just a big lookup table of shape (vocab_size, embed_dim)
word_embedder = nn.Embedding(vocab_size, embed_dim)

# Suppose our sentence "bring me the rice chips" tokenized to [0, 1, 2, 3, 4]
token_ids = torch.tensor([[0, 1, 2, 3, 4]])  # shape: (1, 5)

word_embeds = word_embedder(token_ids)
print(word_embeds.shape)  # torch.Size([1, 5, 4096])
                          # (batch=1, sequence_length=5, embed_dim=4096)

# Each of the 5 tokens is now a 4096-dimensional vector.
# The Transformer processes these 5 vectors together using attention.
The key insight for PaLM-E

The Transformer LLM only ever sees vectors of size embed_dim. It doesn't care if a vector came from a word token or from an image encoder — as long as it's the right size. PaLM-E exploits this to inject images!

👁️ Concept 2: The Vision Transformer (ViT)

PaLM-E uses a Vision Transformer (ViT) to turn images into sequences of vectors — the same shape as word embeddings.

How ViT turns an image into tokens

Original image: 224 × 224 pixels ┌────────────────────────────────────────────────┐ │ ░░░░░░░░░░░░ ▓▓▓▓▓▓▓▓▓▓▓▓ ▒▒▒▒▒▒▒▒▒▒▒▒ ... │ │ ░░░░░░░░░░░░ ▓▓▓▓▓▓▓▓▓▓▓▓ ▒▒▒▒▒▒▒▒▒▒▒▒ ... │ │ ... │ └────────────────────────────────────────────────┘ │ │ Step 1: Cut into 16×16 pixel patches ▼ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ... │P1│ │P2│ │P3│ │P4│ │P5│ │P6│ │P7│ 196 patches total └──┘ └──┘ └──┘ └──┘ └──┘ └──┘ └──┘ (224/16 = 14, so 14×14 = 196) │ │ Step 2: Flatten each patch → vector of size 768 │ (16×16×3 = 768 for RGB) │ │ Step 3: Linear projection → embed_dim vectors │ │ Step 4: Run through Transformer layers ▼ [v1] [v2] [v3] [v4] ... PYTORCH — Simplified ViT patch embedding
import torch
import torch.nn as nn

class PatchEmbedding(nn.Module):
    """
    Turns an image into a sequence of patch embeddings.
    This is the first step of a Vision Transformer.
    """
    def __init__(self, img_size=224, patch_size=16, in_channels=3, embed_dim=768):
        super().__init__()
        self.num_patches = (img_size // patch_size) ** 2  # 196

        # A Conv2d with kernel=patch_size, stride=patch_size is a neat trick:
        # it extracts non-overlapping patches AND projects them to embed_dim
        # in one operation. Same as "cut into patches then linear project".
        self.proj = nn.Conv2d(in_channels, embed_dim,
                              kernel_size=patch_size, stride=patch_size)

    def forward(self, x):
        # x shape: (batch, 3, 224, 224)
        x = self.proj(x)          # → (batch, embed_dim, 14, 14)
        x = x.flatten(2)         # → (batch, embed_dim, 196)
        x = x.transpose(1, 2)    # → (batch, 196, embed_dim)
        return x                  # 196 vectors, each of size embed_dim

# --- Try it ---
patcher = PatchEmbedding(img_size=224, patch_size=16, embed_dim=768)
fake_image = torch.randn(1, 3, 224, 224)  # 1 random image
patches = patcher(fake_image)
print(patches.shape)  # torch.Size([1, 196, 768])
                       # → 196 image tokens, each 768-dim
                       # Exactly like having 196 "words" from the image!

In PaLM-E, the ViT used is the ViT-22B (22 billion parameters!), producing high-quality semantic patch embeddings. After the ViT, a small linear projection maps from ViT's embedding dimension to PaLM's embedding dimension.

The Three Encoder Types in PaLM-E

1

State Vector Encoder (MLP)

For robot joint angles, object poses, etc. A simple MLP maps the raw state vector (e.g. 50 numbers) into a few embedding vectors. Simplest and fastest.

2

Vision Transformer (ViT)

For 2D camera images. The ViT breaks the image into patches and runs Transformer attention over them. PaLM-E uses ViT-4B or ViT-22B pre-trained on ImageNet classification. A linear layer projects ViT's output dim to PaLM's dim.

3

OSRT — Object Scene Representation Transformer

A 3D-aware encoder. Takes multiple views of a scene, discovers individual objects automatically (no manual labeling!), and outputs one embedding vector per object. Great for manipulation — the robot needs to know WHERE objects are in 3D space.

🧩 Concept 4: Multimodal Sentences (The Core Idea)

This is the paper's central trick. Let's implement it step by step.

The math (plain English version)

Normally, in an LLM, each position i in the input sequence has an embedding vector x_i that comes from the word embedding table:

x_i = word_embedder(token_id_i)   # a vector of size embed_dim

PaLM-E generalises this: some positions can use a different source:

x_i = word_embedder(token_id_i)   # if position i is a text token
x_i = image_encoder(image_j)[k]   # if position i is the k-th patch of image j
x_i = state_encoder(state_j)[k]   # if position i is part of robot state j

The Transformer LLM never knows the difference — it just sees a sequence of same-size vectors.

PYTORCH — Building a multimodal sentence
import torch
import torch.nn as nn

# ── Toy dimensions (real PaLM-E uses 4096+) ──
EMBED_DIM  = 64    # dimension every token/patch/state vector must have
VOCAB_SIZE = 1000  # toy vocabulary
IMG_PATCHES = 16  # toy: 4x4 patches instead of 14x14

# ── The three encoders ──────────────────────────────────────────────────

# 1) Word embedding table (the normal LLM part)
word_embedder = nn.Embedding(VOCAB_SIZE, EMBED_DIM)

# 2) Image encoder: ViT produces patch embeddings, then project to EMBED_DIM
class TinyImageEncoder(nn.Module):
    def __init__(self):
        super().__init__()
        # In real PaLM-E: ViT-22B produces 1024-dim patches,
        # then a linear layer projects to PaLM's 4096-dim.
        # Here: fake image → 16 patches, each EMBED_DIM.
        self.patch_proj = nn.Linear(3 * 8 * 8, EMBED_DIM)  # 8x8 RGB patch

    def forward(self, img):
        # img: (batch, 3, 32, 32)  → 16 patches of 8×8
        B = img.shape[0]
        # Manually extract 4×4 grid of 8×8 patches
        patches = img.unfold(2, 8, 8).unfold(3, 8, 8)  # (B, 3, 4, 4, 8, 8)
        patches = patches.reshape(B, 3, 16, 64)            # (B, 3, 16, 64)
        patches = patches.permute(0, 2, 1, 3).reshape(B, 16, -1) # (B, 16, 192)
        return self.patch_proj(patches)                      # (B, 16, EMBED_DIM)

# 3) State encoder: robot state vector → a couple of embedding vectors
class StateEncoder(nn.Module):
    def __init__(self, state_dim=10, num_tokens=4):
        super().__init__()
        self.mlp = nn.Sequential(
            nn.Linear(state_dim, EMBED_DIM * num_tokens),
            nn.ReLU(),
        )
        self.num_tokens = num_tokens

    def forward(self, state):
        # state: (batch, state_dim)
        B = state.shape[0]
        out = self.mlp(state)                    # (B, EMBED_DIM * num_tokens)
        return out.view(B, self.num_tokens, EMBED_DIM)  # (B, 4, EMBED_DIM)

# ── Now BUILD a multimodal sentence ──────────────────────────────────────
#
# Prompt template (as the paper describes):
# "Human: Bring me rice chips. Robot: I see <IMAGE> . State: <STATE> . Next step:"
#
# After tokenizing the text parts (ignoring <IMAGE>/<STATE> placeholders):
# text_before_img = [42, 7, 3, 10, 5]     → 5 text tokens
# text_after_img  = [8, 2, 9]             → 3 more text tokens
# image = random 32x32 RGB image
# state = random 10-dim robot state

img_enc   = TinyImageEncoder()
state_enc = StateEncoder()

B = 1  # batch size 1
text_before = torch.randint(0, VOCAB_SIZE, (B, 5))   # 5 text tokens
text_after  = torch.randint(0, VOCAB_SIZE, (B, 3))   # 3 text tokens
image       = torch.randn(B, 3, 32, 32)               # a 32×32 RGB image
state       = torch.randn(B, 10)                      # robot state vector

# Encode each modality into vectors of size EMBED_DIM
text_before_emb = word_embedder(text_before)   # (1, 5, 64)
text_after_emb  = word_embedder(text_after)    # (1, 3, 64)
image_emb       = img_enc(image)               # (1, 16, 64)
state_emb       = state_enc(state)             # (1, 4, 64)

# ★ THE KEY STEP: concatenate along the sequence dimension ★
multimodal_sequence = torch.cat([
    text_before_emb,   # 5 tokens  — "Human: Bring me rice chips..."
    image_emb,         # 16 tokens — image patches injected HERE
    state_emb,         # 4 tokens  — robot state injected HERE
    text_after_emb,    # 3 tokens  — ". Next step:"
], dim=1)

print(multimodal_sequence.shape)
# torch.Size([1, 28, 64])
# ↑ 28 = 5 text + 16 image patches + 4 state tokens + 3 text
# The Transformer LLM will process all 28 vectors identically.
# It has NO idea which came from text vs image vs state!
Why is this beautiful?

The LLM's self-attention mechanism lets every token attend to every other token. So a text token like "chips" can directly attend to the image patch that shows the chips bag. The model learns to associate visual features with language concepts automatically through training.

🏋️ Concept 5: How PaLM-E is Trained

The Loss Function: Cross-Entropy on the Output Text

PaLM-E is a decoder-only model — it generates the answer token by token. The training signal is simple: predict the next token correctly. The loss only applies to the output tokens (the answer), not the prompt.

Training example: ───────────────────────────────────────────────────────────── PREFIX (given as input — no loss computed on these): "Human: Bring me rice chips. Robot: I see [IMAGE]. Step:" TARGET (model must predict these tokens — loss computed here): "1. Go to drawers. 2. Open top drawer. 3. Pick green bag." Loss = average cross-entropy over each target token = -( log P("1") + log P("Go") + log P("to") + ... ) / N In words: for each position in the target, "how surprised was the model by the correct next token?" The more surprised, the higher the loss. Training adjusts weights to reduce this surprise.
PYTORCH — Training loop (simplified)
import torch
import torch.nn as nn

class TinyPaLME(nn.Module):
    """
    A tiny, simplified PaLM-E.
    Real PaLM-E has 562B params; this has ~a few thousand.
    Same IDEAS, different scale.
    """
    def __init__(self, vocab_size=1000, embed_dim=64, nhead=4, num_layers=2):
        super().__init__()

        # ── Encoders ──────────────────────────────────
        self.word_embedder = nn.Embedding(vocab_size, embed_dim)
        # (In real PaLM-E, ViT-22B would be here instead of this tiny linear)
        self.img_proj = nn.Linear(192, embed_dim)  # 8×8×3 patch → embed_dim

        # ── The LLM (decoder-only Transformer) ────────
        decoder_layer = nn.TransformerDecoderLayer(
            d_model=embed_dim, nhead=nhead, dim_feedforward=embed_dim*4,
            batch_first=True
        )
        self.transformer = nn.TransformerDecoder(decoder_layer, num_layers=num_layers)

        # ── Output head: project back to vocabulary ───
        self.output_head = nn.Linear(embed_dim, vocab_size)

    def forward(self, text_tokens, image_patches, target_tokens):
        # text_tokens:   (B, T_text) — prefix text token IDs
        # image_patches: (B, 16, 192) — pre-extracted 8×8 RGB patches
        # target_tokens: (B, T_target) — answer token IDs

        B, T_text = text_tokens.shape
        _, T_target = target_tokens.shape

        # 1) Embed each modality
        text_emb  = self.word_embedder(text_tokens)     # (B, T_text, D)
        img_emb   = self.img_proj(image_patches)         # (B, 16, D)
        target_emb = self.word_embedder(target_tokens)   # (B, T_target, D)

        # 2) Build multimodal prefix: [text | image patches]
        prefix = torch.cat([text_emb, img_emb], dim=1)  # (B, T_text+16, D)

        # 3) Causal mask for autoregressive generation
        # (each target token can only attend to previous target tokens)
        causal_mask = nn.Transformer.generate_square_subsequent_mask(T_target)

        # 4) Decoder: target attends to itself (causally) and to the prefix
        out = self.transformer(
            tgt=target_emb,     # what we're generating
            memory=prefix,      # the multimodal context (text + image)
            tgt_mask=causal_mask
        )  # (B, T_target, D)

        # 5) Project to vocabulary logits
        logits = self.output_head(out)  # (B, T_target, vocab_size)
        return logits


# ── Training step ─────────────────────────────────────────────────────
model     = TinyPaLME()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-4)
loss_fn   = nn.CrossEntropyLoss()

# Fake batch of data
B = 4
text_tokens    = torch.randint(0, 1000, (B, 8))   # 8-token prompt
image_patches  = torch.randn(B, 16, 192)           # 16 patches × 192-dim
target_tokens  = torch.randint(0, 1000, (B, 12))  # 12-token answer

optimizer.zero_grad()

# Input to the model: tokens 0..10; Target: tokens 1..11 (next-token prediction)
logits = model(text_tokens, image_patches, target_tokens[:, :-1])
# logits: (B, 11, 1000)

# Reshape for loss: (B*11, 1000) vs (B*11,)
loss = loss_fn(
    logits.reshape(-1, 1000),
    target_tokens[:, 1:].reshape(-1)
)

loss.backward()
optimizer.step()
print(f"Loss: {loss.item():.4f}")  # e.g. Loss: 6.9077

What gets trained and what gets frozen?

Option A: Freeze the LLM

Only train the image/state encoders + the linear projection layers. The LLM stays exactly as pre-trained. Works! But sometimes struggles on robotics tasks because the LLM can't adapt its internal representations.

Option B: Train Everything (Finetune LLM)

Train all weights end-to-end. Better performance but risks catastrophic forgetting — the model forgets how to do language tasks while learning robotics. The paper shows larger models forget less.

What is Catastrophic Forgetting?

Imagine you're a world-class English speaker. You start learning Japanese intensively. If you study nothing but Japanese for months, you might start to forget some nuanced English. Neural networks have the same problem — when you train on new data, old knowledge can get overwritten. PaLM-E-562B only loses 3.9% of its language ability, while PaLM-E-12B loses 87.3%!

Co-training: The Secret Sauce

PaLM-E isn't just trained on robot data. It's trained on a huge mixture:

Training Data Mixture: ════════════════════════════════════════════ Internet-scale vision+language data ~91.1% ├── WebLI (web images + captions) ├── VQA datasets (image + question + answer) ├── COCO captions └── ... Robotics data ~8.9% ├── TAMP manipulation tasks ├── Language-Table pushing tasks └── SayCan mobile manipulation ════════════════════════════════════════════ KEY FINDING: Training on the full mixture makes the robot tasks work BETTER than training on robot data alone! This is called "positive transfer" — knowledge from one task helps performance on another.

palme_demo.py — Complete runnable code
"""
PaLM-E Concept Demo
===================
A simplified but faithful implementation of PaLM-E's core ideas.
Run with: python palme_demo.py

What this demos:
 1. Text tokenization and embedding
 2. Image patch encoding (mini ViT-style)
 3. Robot state encoding
 4. Building multimodal sentences
 5. Decoder-only LLM forward pass
 6. Auto-regressive text generation
 7. Connecting output to robot actions
"""

import torch
import torch.nn as nn
import torch.nn.functional as F
import math

# ═══════════════════════════════════════════════════════════════
# CONSTANTS
# ═══════════════════════════════════════════════════════════════
VOCAB_SIZE   = 512    # toy vocabulary (real PaLM: 256k)
EMBED_DIM    = 128   # vector size (real PaLM-E: 4096+)
N_HEADS      = 4     # attention heads (real: 64+)
N_LAYERS     = 3     # transformer layers (real: 100+)
IMG_PATCHES  = 16    # image tokens per image (real ViT-22B: 256+)
STATE_DIM    = 6     # robot state: [x, y, z, rx, ry, rz]
STATE_TOKENS = 2     # state encoded into 2 tokens
MAX_GEN      = 30    # max tokens to generate

# Simple toy vocabulary for the demo
WORD_TO_ID = {
    "<pad>":0, "<bos>":1, "<eos>":2,
    "human":3, "bring":4, "me":5, "rice":6, "chips":7,
    "robot":8, "i":9, "see":10, "step":11, "go":12,
    "to":13, "drawers":14, "open":15, "drawer":16,
    "pick":17, "green":18, "bag":19, "terminate":20,
}
ID_TO_WORD = {v: k for k, v in WORD_TO_ID.items()}

def tokenize(sentence):
    """Convert a sentence to token IDs."""
    return [WORD_TO_ID.get(w, 0) for w in sentence.lower().split()]

def detokenize(ids):
    """Convert token IDs back to words."""
    return " ".join(ID_TO_WORD.get(i, "<unk>") for i in ids)


# ═══════════════════════════════════════════════════════════════
# ENCODERS
# ═══════════════════════════════════════════════════════════════

class ImageEncoder(nn.Module):
    """
    Simplified ViT-style image encoder.
    Real PaLM-E: ViT-22B (22 billion params!)
    Here: tiny MLP for concept demonstration.
    Takes a fake "image feature" vector and projects to IMG_PATCHES tokens.
    """
    def __init__(self):
        super().__init__()
        # In reality: ViT extracts 196 patches from a 224×224 image.
        # Here: we pretend image_features has size 256 (like 16 patches × 16-dim)
        self.patch_proj = nn.Linear(16, EMBED_DIM)   # project each fake patch
        # Linear projection from ViT's dim to LLM's dim (paper calls this ψ)
        self.llm_proj = nn.Linear(EMBED_DIM, EMBED_DIM)

    def forward(self, image_features):
        # image_features: (B, IMG_PATCHES, 16) — fake patch features
        x = self.patch_proj(image_features)   # (B, IMG_PATCHES, EMBED_DIM)
        x = F.gelu(x)                         # nonlinearity
        x = self.llm_proj(x)                   # (B, IMG_PATCHES, EMBED_DIM)
        return x                               # ready to mix with text tokens


class StateEncoder(nn.Module):
    """
    Encodes a robot state vector into STATE_TOKENS embedding vectors.
    Real PaLM-E: MLP φ_state.
    """
    def __init__(self):
        super().__init__()
        self.mlp = nn.Sequential(
            nn.Linear(STATE_DIM, EMBED_DIM * STATE_TOKENS),
            nn.ReLU()
        )

    def forward(self, state):
        # state: (B, STATE_DIM)
        B = state.shape[0]
        out = self.mlp(state)              # (B, EMBED_DIM * STATE_TOKENS)
        return out.view(B, STATE_TOKENS, EMBED_DIM)  # (B, 2, EMBED_DIM)


# ═══════════════════════════════════════════════════════════════
# POSITIONAL ENCODING
# ═══════════════════════════════════════════════════════════════

class PositionalEncoding(nn.Module):
    """
    Adds position information to embeddings.
    Without this, the Transformer doesn't know the ORDER of tokens.
    Uses sin/cos waves of different frequencies — a clever trick
    where each position gets a unique "fingerprint" of wave values.
    """
    def __init__(self, max_len=512):
        super().__init__()
        pe = torch.zeros(max_len, EMBED_DIM)
        pos = torch.arange(max_len).unsqueeze(1)            # (max_len, 1)
        div = torch.exp(torch.arange(0, EMBED_DIM, 2).float() *
                        (-math.log(10000.0) / EMBED_DIM))   # frequency terms
        pe[:, 0::2] = torch.sin(pos * div)  # even dims: sin
        pe[:, 1::2] = torch.cos(pos * div)  # odd dims:  cos
        self.register_buffer('pe', pe.unsqueeze(0))  # (1, max_len, EMBED_DIM)

    def forward(self, x):
        # x: (B, T, EMBED_DIM)
        return x + self.pe[:, :x.shape[1], :]


# ═══════════════════════════════════════════════════════════════
# THE LLM (decoder-only Transformer)
# ═══════════════════════════════════════════════════════════════

class PaLMEModel(nn.Module):
    """
    The full PaLM-E model (simplified).

    Architecture:
      Image Encoder + State Encoder + Word Embedder
        → multimodal sequence
        → Positional Encoding
        → Decoder-only Transformer (causal attention)
        → Linear head → logits over vocabulary
        → Sample/argmax → next token
    """
    def __init__(self):
        super().__init__()

        # Modality encoders
        self.word_embedder = nn.Embedding(VOCAB_SIZE, EMBED_DIM, padding_idx=0)
        self.img_encoder   = ImageEncoder()
        self.state_encoder = StateEncoder()
        self.pos_enc       = PositionalEncoding()

        # The LLM: stack of Transformer decoder layers
        # Each layer: self-attention (causal) + cross-attention + FFN
        # Note: PaLM uses a single-stack (no cross-attention to a separate encoder)
        # We use TransformerEncoder here (all self-attention, causal mask applied)
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=EMBED_DIM,
            nhead=N_HEADS,
            dim_feedforward=EMBED_DIM * 4,
            dropout=0.0,
            batch_first=True,
            norm_first=True   # Pre-LN (used in PaLM for stability)
        )
        self.transformer = nn.TransformerEncoder(encoder_layer, num_layers=N_LAYERS)

        # Output projection: hidden state → logits over vocabulary
        self.output_head = nn.Linear(EMBED_DIM, VOCAB_SIZE, bias=False)

        # Tie weights: output_head shares weights with word_embedder
        # (a common trick that improves quality and reduces parameters)
        self.output_head.weight = self.word_embedder.weight

    def build_multimodal_prefix(self, text_ids, image_feats, state):
        """
        Build the multimodal prefix sequence.
        text_ids:    (B, T_text)
        image_feats: (B, IMG_PATCHES, 16) or None
        state:       (B, STATE_DIM) or None
        Returns: (B, T_total, EMBED_DIM)
        """
        parts = []

        # Text part
        parts.append(self.word_embedder(text_ids))    # (B, T_text, D)

        # Image part (injected WITHIN the text sequence)
        if image_feats is not None:
            parts.append(self.img_encoder(image_feats))  # (B, 16, D)

        # Robot state part
        if state is not None:
            parts.append(self.state_encoder(state))   # (B, 2, D)

        prefix = torch.cat(parts, dim=1)             # (B, T_total, D)
        prefix = self.pos_enc(prefix)                 # add positional encoding
        return prefix

    def forward(self, text_ids, image_feats, state, target_ids):
        """
        Full forward pass for training.
        Returns logits: (B, T_target, vocab_size)
        """
        B, T_target = target_ids.shape

        # 1) Build multimodal prefix
        prefix = self.build_multimodal_prefix(text_ids, image_feats, state)

        # 2) Embed target tokens
        target_emb = self.word_embedder(target_ids)  # (B, T_target, D)
        target_emb = self.pos_enc(target_emb)

        # 3) Concatenate prefix + target
        full_seq = torch.cat([prefix, target_emb], dim=1)
        T_full = full_seq.shape[1]

        # 4) Causal mask: each position can only attend to positions before it
        causal_mask = torch.triu(
            torch.ones(T_full, T_full), diagonal=1
        ).bool()
        # True = "this position is masked (ignored)"

        # 5) Run through Transformer
        out = self.transformer(full_seq, mask=causal_mask, is_causal=True)

        # 6) Only take the output at target positions
        T_prefix = prefix.shape[1]
        target_out = out[:, T_prefix:, :]   # (B, T_target, D)

        # 7) Project to vocabulary
        logits = self.output_head(target_out)  # (B, T_target, VOCAB_SIZE)
        return logits

    @torch.no_grad()
    def generate(self, text_ids, image_feats, state, max_new_tokens=20):
        """
        Auto-regressively generate a plan given the multimodal context.
        This is how PaLM-E generates robot instructions at test time.
        """
        self.eval()
        B = text_ids.shape[0]

        # Start with <bos> token
        generated = torch.tensor([[WORD_TO_ID['<bos>']]] * B)

        for step in range(max_new_tokens):
            # Run forward pass on everything generated so far
            logits = self.forward(text_ids, image_feats, state, generated)

            # Take the last position's logits (predict NEXT token)
            next_logits = logits[:, -1, :]   # (B, VOCAB_SIZE)

            # Sample from the distribution (temperature=1.0)
            # In practice, beam search or greedy decoding is often used
            probs = torch.softmax(next_logits, dim=-1)
            next_token = torch.multinomial(probs, num_samples=1)  # (B, 1)

            # Append the new token
            generated = torch.cat([generated, next_token], dim=1)

            # Stop if EOS token generated
            if (next_token == WORD_TO_ID['<eos>'']).all():
                break

        return generated[:, 1:]  # strip the <bos> token

🎯 Key Takeaways

1

Inject, Don't Describe

Instead of converting images to text descriptions, inject raw image embeddings directly into the LLM's token stream. The model learns its own visual language internally.

2

Positive Transfer is Real

Training on diverse internet-scale vision+language data makes the model BETTER at robot tasks, even though only 8.9% of training data is robotics data.

3

Scale Beats Forgetting

Larger models retain original language capabilities better when fine-tuned on new tasks. PaLM-E-562B forgets only 3.9% of language ability vs 87.3% for the 12B model.

4

3D Representations Win

OSRT (3D-aware object-centric encoder) outperforms standard ViT for manipulation tasks. When robots need to know WHERE things are in 3D space, geometry matters.

5

One Model, Many Tasks

A single PaLM-E-562B checkpoint handles VQA, captioning, robot manipulation, affordance prediction, AND failure detection. Generalist > specialist.

6

Freezing is a Viable Path

You can keep the LLM frozen and only train the vision encoder + projector layer. This prevents catastrophic forgetting entirely, at some cost to robot performance.

The Big Picture

PaLM-E is essentially asking: "What if a robot's brain was a language model that could also see and feel?" By treating sensor data as just another type of "token", the robot inherits all the world knowledge, reasoning ability, and language understanding of a massive LLM — and just needs to learn how to "translate" pixels and sensor readings into that shared embedding space. The result is a robot that can understand nuanced human instructions, reason about complex multi-step plans, and even crack jokes about what it sees in a camera.
