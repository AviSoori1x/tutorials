01 — Root Problem

The exploration
bottleneck

Reinforcement Learning for LLMs has a fundamental chicken-and-egg problem. RL improves a model by learning from high-quality samples — but to get high-quality samples, the model must first be able to generate them. If a model never stumbles onto a good reasoning path, it never gets a reward signal for it, and it never learns it.

The Core Loop

bad model → generates bad outputs → RL learns from bad outputs → stays bad. Policy entropy gradually collapses during training, causing the model to converge toward a limited set of reasoning trajectories. The inability to explore restricts the ability to learn.

Prior work attempted to fix this through entropy-based exploration bonuses (EA-RL), hybrid-policy sampling (RL-Plus), or prolonged training (ProRL). But these approaches largely bias the policy distribution toward high-reward responses already supported by base models, rather than truly expanding reasoning boundaries. They explore within the existing distribution, not beyond it.

Standard RLVR — Fails Here

Works for verifiable domains

  • Math: final answer is right/wrong
  • Code: tests pass/fail
  • Single binary reward — clear, dense
  • Entropy collapses → same reasoning paths
  • Can't explore solutions it's never generated
Open-Ended Tasks — Gap

Medicine, writing, instruction following

  • No single ground truth answer
  • Multi-dimensional quality (safety, completeness)
  • LLM-as-judge: noisy, biased
  • Random exploration finds garbage
  • No structured path to quality solutions

02 — Core Idea

RuscaRL in one sentence

"Use rubrics not just to evaluate answers, but to guide the model's exploration during training — then gradually remove that guidance so the model internalizes better reasoning."

RuscaRL leverages rubrics in two complementary ways that run simultaneously:

1. Rubric Generation

For each training prompt, generate a checklist rubric (from HealthBench, or via GPT-4.1 for other domains). Rubric = criteria with +/− points.

2. Scaffolded Rollout

Sample G=8 responses. Each gets a different fraction of rubric criteria injected into the prompt (100%, 86%, 71%… down to 0%). Different levels → diverse outputs.

3. Rubric-Based Reward

Grader LLM (Qwen3-32B) evaluates each response against ALL rubric criteria — each criterion scored binary 0/1. Weighted sum gives scalar reward.

4. GRPO Update

Group-relative advantage estimation. Policy updated to increase probability of above-average responses. Log-prob computed WITHOUT scaffold (πθ(o|q)) to encourage internalization.

5. Decay Scaffolding

After each training step, the sigmoid decay function reduces the scaffolding ratio λ_step(t). As training progresses, less and less rubric guidance is injected until it reaches 0.

03 — Foundation

Rubric-based
evaluation system

A rubric R = {c₁, c₂, …, c_N} is a set of N verifiable criteria. Each criterion c_i has a clear description and a points value p_i indicating its contribution. Points can be positive (good behaviors to reward) or negative (bad behaviors to penalize).

R = {c₁, c₂, …, c_N}
with point vector p = [p₁, p₂, …, p_N]

The grader function G (implemented by an LLM) takes the judge prompt and outputs a binary decision:

b_i = G(q, o, c_i) ∈ {0, 1}
binary indicator: criterion c_i is satisfied (1) or not (0)

Extending to the full rubric produces a binary indicator vector, then element-wise multiplication with points:

s = b ⊙ p = [b₁p₁, b₂p₂, …, b_N p_N]
fine-grained score vector across all criteria
Example rubric — "Introduce the idea of reinforcement learning"
+pts
Step-by-step analysis with complete structural logic. Each conceptual step follows from the previous with clear connective reasoning.
+3
+pts
Explains clearly key elements: agent, environment, reward. All three must be defined with correct roles and relationships.
+6
−pts
Confuses environment and reward roles in the explanation. For instance, describing the environment as that which assigns scalar feedback.
−7

The total possible score S_total = Σ p_j where the sum is over all positive-point criteria only (M items). This is used for normalization to keep the final reward in a comparable range.

Key Design Detail — Negative Points

Negative criteria (pitfalls) are phrased as undesirable behaviors. The grader returns whether the criterion IS met (even when it's undesirable). A good response does NOT meet negative criteria, so its score for those items is 0 (not penalized). A bad response DOES meet the negative criterion and gets penalized. This is the same convention as HealthBench.

04 — Key Innovation

Rubric-based scaffolding
for RL exploration

The paper draws from Vygotsky's Zone of Proximal Development in educational psychology. When learners' capabilities are insufficient, they benefit from structured guidance to bridge the gap between current ability and target performance. As competence grows, this scaffolding should be gradually withdrawn to foster independent problem-solving (Wood et al., 1976).

The rubric-based scaffolding mechanism augments the original policy function by adding a subset of rubric criteria R_S as additional guidance. The policy becomes:

π_θ(o | q, R_S)
policy conditioned on instruction q AND rubric subset R_S

The scaffolding ratio λ_S controls how many criteria are included:

|R_S| = round(λ_S × |R|)
sample this many criteria from the full rubric set R

There are two dimensions to control λ_S — and their product gives the final ratio for each sample:

λ_{S,i} = λ_step(t) × λ_i
Integrated scaffolding ratio: inter-step decay × intra-group differentiation
Scaffolding Prompt Template (Appendix E.2)

The prompt prefixes the instruction with: "IMPORTANT POINTS TO INCLUDE (you should aim to address these): [criteria…]" and "IMPORTANT POINTS TO AVOID (you should not do these): [criteria…]". Critically, the model is told: "Do not mention or reference these evaluation criteria in your response. Your response should appear natural and spontaneous. Revealing that you have access to evaluation criteria would be considered cheating."

05 — Dimension 1

Intra-group scaffolding
differentiation

GRPO requires response diversity within each sampling group to avoid collapse into homogeneous samples (which would make all advantages ≈ 0, killing the learning signal). To guarantee diversity, each sample in a group of G gets a different amount of rubric guidance.

λ_i = (G − i) / (G − 1)
for i = 1, 2, …, G — linearly decreasing from 100% to 0%

So with G=8 samples:

06 — Dimension 2

Inter-step scaffolding
decay

The intra-group differentiation handles diversity within a training step. The inter-step decay handles removing guidance across time. As the model develops independent reasoning strategies, external hints are gradually withdrawn:

λ_step(t) = 1 / (1 + e^{α(t − t₀)})
t ∈ [0,1] normalized training progress · t₀ = midpoint of decay · α = steepness

Optimal hyperparameters found by ablation: α = 125, t₀ = 0.2 for most models. This means the main decay happens early in training (around 20% through), ensuring substantial guidance at the start and near-zero guidance by the end.

Why Sigmoid Wins

Linear and power decay functions perform poorly because prolonged scaffolding causes the model to overfit to the scaffolding cues rather than learning the underlying task. The sigmoid function's smooth S-curve provides adequate support early, then removes it rapidly at the midpoint, then stays near zero — avoiding both prolonged dependence and abrupt removal.

07 — Reward Design

Rubric-based reward
aggregation

The multi-dimensional score vector s = [s₁, s₂, …, s_N] is aggregated into a scalar reward by summing and normalizing:

S = Σᵢ sᵢ / S_total
S_total = sum of all positive-point criteria scores · reward r_i = S_i

This produces scores that fall within [0,1] in most cases, with occasional negative scores possible (if many negative criteria are triggered). The reward is used directly: r_i = S_i.

Critical Detail — Importance Sampling Without Scaffold

During training, the log-probability is computed as π_θ(o_i,t | q, o_i,<t) — WITHOUT the scaffold R_S — even though the response was generated with scaffolding. This acts as a proximal policy update toward a no-scaffold policy, encouraging the model to internalize the reasoning patterns. The theoretically "correct" importance ratio that includes R_S in the denominator (Equation 7) actually performs worse in practice due to high variance from the conditioning mismatch. See §13 for the full importance sampling analysis.

08 — RL Algorithm

GRPO — group relative
policy optimization

RuscaRL uses GRPO (Group Relative Policy Optimization) as its core RL algorithm. Unlike PPO, GRPO eliminates the need for a value model by using group-based advantage estimation — comparing responses within a sampled group rather than against a learned baseline.

For each instruction q, sample G responses and maximize:

J_GRPO(θ) = E[ (1/G) Σᵢ (1/|oᵢ|) Σₜ min(ρᵢ,ₜ(θ)Âᵢ, clip(ρᵢ,ₜ, 1−ε, 1+ε)Âᵢ) ]
ρᵢ,ₜ(θ) = πθ(oᵢ,ₜ|q,oᵢ,<ₜ) / πθ_old(oᵢ,ₜ|q,oᵢ,<ₜ) — token-level importance ratio

The group-relative advantage is computed as:

Â_i = (r_i − mean{r_j}) / std{r_j}
normalized within each group — better-than-average → positive advantage
Why GRPO Works Well With Scaffolding

GRPO's group-relative advantage requires diversity in the group to produce non-trivial advantages. If all G responses are identical, mean = every response, std ≈ 0, and all advantages collapse to 0/0. The intra-group scaffolding differentiation directly solves this: it guarantees structural diversity (highly-guided vs. free responses) which produces meaningful reward variance and non-degenerate advantages.

09 — Pseudocode

Complete algorithm
pseudocode

Input: Policy model π_θ, data distribution D, grader model G Initialize: Reference policy π_ref ← π_θ   for each training iteration t do for each (q, R) ~ D do // Step 1: Compute scaffolding ratio vector λ_S = λ_step(t) × λ_group // sigmoid decay × linear group   // Step 2: Generate G responses with varying scaffolding for i = 1 to G do R_S ⊂ R // sample λ_{S,i} fraction of criteria o_i ~ π_θ(· | q, R_S) // generate WITH scaffold end for   // Step 3: Evaluate each response against FULL rubric for each response o_i do b = G(q, o_i, R) // binary vector from grader LLM s = b ⊙ p // element-wise with point weights S_i = Σ s_j / S_total // normalized reward end for   // Step 4: GRPO advantage + policy update Â_i = (S_i − mean(S)) / std(S) Compute importance ratio WITHOUT scaffold: ρ_t = π_θ(o_t|q,o_<t) / π_θ_old(o_t|q,o_<t) Update π_θ // maximize clipped PPO objective end for Update scaffolding step ratio λ_step(t) end for return Trained policy π_θ

Policy Entropy During Training

Exploration-Exploitation Balance

RuscaRL's entropy curve shows a characteristic shape: it rises early as scaffolding helps the model explore diverse reasoning trajectories, then declines as scaffolding is removed and the model converges to high-quality learned patterns. Rubric-based RL shows only monotonic entropy collapse. This S-shaped trajectory is the hallmark of scaffolding working as intended — explore first, then exploit.

Best-of-N Evaluation

At N=2048 (upper bound on model's reasoning capability), RuscaRL outperforms both the initial model and Rubric-based RL — validating that it genuinely expands the reasoning boundary, not just improves average quality. At N=1, it also shows better single-sample quality, indicating improved reasoning stability.

13 — Theoretical Detail

Importance sampling
analysis

This section addresses a subtle but important question: when the behavior policy uses scaffold R_S but the target policy does not, what is the correct importance ratio?

ρ_correct = π_θ(o_t|q,o_<t) / π_θ_old(o_t|q,R_S,o_<t)
Equation 7 — theoretically correct unbiased IS ratio for no-scaffold target

However, the paper uses a different ratio in practice:

ρ_used = π_θ(o_t|q,o_<t) / π_θ_old(o_t|q,o_<t)
Not theoretically correct IS — but acts as proximal update toward no-scaffold reference policy
Importance Ratio Method Comparison
ρ MethodHealthBench-500LLMEval-MedNotes
π_θ(o|q) / π_θold(o|q) — used in paper56.465.3Best practical performance
π_θ(o|q) / π_θold(o|q,R_S) — theoretically correct44.853.8High variance, unstable
π_θ(o|q,R_S) / π_θold(o|q,R_S) — consistent conditioning45.755.7Doesn't push toward no-scaffold
The Practical vs Theoretical Tradeoff

The "wrong" importance ratio (Method 1) outperforms the theoretically correct one (Method 2) because Method 2 suffers from high variance due to the conditioning mismatch between numerator (no scaffold) and denominator (with scaffold). Method 3 maintains theoretical consistency but doesn't encourage scaffold-free reasoning. Method 1 effectively acts as a proximal policy update toward a reference no-scaffold policy — it pushes the model to match the scaffold-free distribution while maintaining training stability via the clipping mechanism.

15 — Implementation

Code examples —
building RuscaRL

Scaffolding ratio computation — the combined λ_S for all G samples at training step t:

import torch
import math

# ── Intra-group scaffolding ratios ──────────────────────
def intra_group_ratios(G: int) -> list[float]:
    """Linear differentiation: λ_i = (G-i)/(G-1)"""
    return [(G - i) / (G - 1) for i in range(1, G + 1)]

# G=8: [1.0, 0.857, 0.714, 0.571, 0.429, 0.286, 0.143, 0.0]
print(intra_group_ratios(8))

# ── Inter-step decay ────────────────────────────────────
def sigmoid_decay(t: float, alpha: float = 125, t0: float = 0.2) -> float:
    """λ_step(t) = 1 / (1 + exp(α(t - t₀)))"""
    return 1.0 / (1.0 + math.exp(alpha * (t - t0)))

# Early training (t=0.05) → high scaffolding
print(f"t=0.05: λ_step = {sigmoid_decay(0.05):.4f}")  # ≈ 1.0
# At midpoint (t=0.2) → half scaffolding
print(f"t=0.20: λ_step = {sigmoid_decay(0.20):.4f}")  # ≈ 0.5
# Late training (t=0.35) → near zero
print(f"t=0.35: λ_step = {sigmoid_decay(0.35):.4f}")  # ≈ 0.0

# ── Combined scaffolding matrix ─────────────────────────
def scaffolding_matrix(G: int, t: float) -> list[float]:
    """λ_S,i = λ_step(t) × λ_i for all samples in group"""
    step = sigmoid_decay(t)
    group = intra_group_ratios(G)
    return [step * g for g in group]

# Early in training (t=0.05): [1.0, 0.857, 0.714, ..., 0.0]
# Late in training  (t=0.40): [0.0, 0.0,   0.0,   ..., 0.0]

Sampling rubric subsets and building scaffolded prompts:

import random

def sample_rubric_subset(rubric: list, ratio: float) -> list:
    """Sample round(ratio × |R|) criteria from rubric."""
    k = round(ratio * len(rubric))
    if k == 0:
        return []
    return random.sample(rubric, k)

def build_scaffolded_prompt(instruction: str, rubric_subset: list) -> str:
    """Appendix E.2 template: inject criteria as guidance."""
    if not rubric_subset:
        return instruction  # no scaffolding
    
    pos = [c for c in rubric_subset if c['points'] > 0]
    neg = [c for c in rubric_subset if c['points'] < 0]
    
    prompt = "You are a helpful assistant. For this question, please consider:\n\n"
    
    if pos:
        prompt += "IMPORTANT POINTS TO INCLUDE:\n"
        prompt += "\n".join(f"- {c['criterion']}" for c in pos) + "\n\n"
    
    if neg:
        prompt += "IMPORTANT POINTS TO AVOID:\n"
        prompt += "\n".join(f"- {c['criterion']}" for c in neg) + "\n\n"
    
    prompt += "IMPORTANT: Do not mention these criteria in your response.\n\n"
    prompt += instruction
    return prompt

Rubric-based reward computation:

def compute_rubric_reward(binary_decisions: list[bool], points: list[float]) -> float:
    """S = Σ(b_i × p_i) / S_total"""
    S_total = sum(p for p in points if p > 0)  # positive criteria only
    score = sum(b * p for b, p in zip(binary_decisions, points))
    return score / S_total  # can be negative if many pitfalls triggered

# Example: rubric with +3, +6, -7 points
points = [3, 6, -7]
binary = [1, 1, 0]  # satisfied c1, c2, did NOT trigger c3 (pitfall)
reward = compute_rubric_reward(binary, points)
print(f"Reward: {reward:.3f}")  # (3+6) / (3+6) = 1.0

bad_binary = [0, 0, 1]  # missed both good, triggered pitfall
bad_reward = compute_rubric_reward(bad_binary, points)
print(f"Bad reward: {bad_reward:.3f}")  # -7/9 ≈ -0.78

def simulate_ruscarl_step(G: int, t: float, rubric: list, instruction: str):
    """Simulate one RuscaRL training step."""

    lambdas = scaffolding_matrix(G, t)
    
    # 2. Generate G responses with varying scaffolding
    rewards = []
    for i in range(G):
        subset = sample_rubric_subset(rubric, lambdas[i])
        prompt = build_scaffolded_prompt(instruction, subset)
        
        # response = policy_model.generate(prompt)  # real code
        # binary_eval = grader_model.evaluate(response, full_rubric)  # N LLM calls
        
        # Fake evaluation for illustration
        binary = [random.random() > 0.3 - lambdas[i]*0.2 for _ in rubric]
        reward = compute_rubric_reward(binary, [c['points'] for c in rubric])
        rewards.append(reward)
    
    # 3. GRPO advantage computation
    rewards = torch.tensor(rewards)
    advantages = (rewards - rewards.mean()) / (rewards.std() + 1e-8)
    
    # 4. Policy update
    # Note: log-prob computed WITHOUT scaffold (πθ(o|q), not πθ(o|q,R_S))
    # loss = -sum(logprob_no_scaffold * advantages)  # clipped PPO objective
    
    return {
        'scaffolding_ratios': lambdas,
        'rewards': rewards.tolist(),
        'advantages': advantages.tolist(),
        'step_decay': sigmoid_decay(t)
    }

RuscaRL vs RaR —
the progression

These two papers form a natural sequence. RaR (Gunjal et al., 2025) established rubric-based reward functions. RuscaRL explicitly builds on that work and adds the scaffolding dimension.

What RaR Solved

Better reward signal

  • Rubric → structured reward vs vague Likert
  • Instance-specific criteria beat generic ones
  • Smaller judges better with rubric guidance
  • Generalizes RLVR: k=1 criterion = standard RLVR
  • Works for open-ended domains without ground truth
What RuscaRL Added

Better exploration + reward

  • Rubric → guide generation, not just score it
  • Intra-group differentiation → diversity in GRPO
  • Sigmoid decay → internalization, not dependence
  • Importance ratio without scaffold → pushes to independence
  • Mean importance ratio 5424× vs base (RaR doesn't measure this)
The deeper insight: RL doesn't fail because reward is bad — it fails because exploration is bad. Structured exploration > random exploration.
🎓
Educational Psychology Foundation
Vygotsky's Zone of Proximal Development (1978) and Wood et al. (1976) on scaffolding in problem solving — applied directly to RL training. The model is treated as a learner needing structured hints that are gradually withdrawn.
📊
Exploration Quantification
The importance ratio metric (ρ_seq) provides an objective measure of exploration boundary expansion — showing that RuscaRL reaches genuinely novel solution regions rather than redistributing probability within the existing support.
⚠️
Critical Limitations
~2× training cost vs standard RL. Depends on rubric quality (severe corruption still collapses performance). LLM judge bias inherited. Domain-specific datasets required for best results. Rubric generation pipeline is an unsolved bottleneck.
