Step 2 — EnCodec: Audio → Discrete Tokens
Residual Vector Quantization (RVQ) — How audio becomes a token matrix
EnCodec's encoder produces embeddings at 75 Hz (320× downsampling from 24kHz). Each embedding is then quantized through 8 codebooks in series — each one codes the residual error left by the previous.

Step 3 — AR Model: Generate Codebook 1 Tokens
Autoregressive Transformer Decoder Causal Attention SAMPLING
12 layers · 16 heads · d=1024 · FFN=4096 · dropout=0.1
❓ Why AR for codebook 1? We don't know the output length ahead of time — different speakers have different speaking rates. AR generates tokens one-by-one until <EOS>, naturally handling variable-length sequences.

WEIGHT TYING
🔗 Output projection layer shares weights with the acoustic embedding layer — reduces params, improves training efficiency.
⚡ Inference: use SAMPLING, not beam search. Beam search causes infinite repetition loops in this model. Sampling also produces diverse outputs (different rhythm, pace) per run.
OUTPUT: c[:,1] = [c₀,₁ c₁,₁ c₂,₁ ··· c_{T},₁] — length T is variable, determined by when the model emits EOS
Step 4 — NAR Model: Generate Codebooks 2–8
Non-Autoregressive Transformer Bidirectional Attention GREEDY Decode
12 layers · 16 heads · d=1024 · FFN=4096 · AdaLN
❓ Why NAR for codebooks 2–8? Length T is already fixed from the AR step. Running NAR means we predict all T tokens of a codebook in a single forward pass — reducing inference time from O(T) per codebook to O(1). Seven codebooks × O(1) = fast.
STAGE LOOP: j = 2, 3, 4, 5, 6, 7, 8
At each stage j, inputs are:
Phonemes x [L] + Full C̃ [T₀×8]
ALL 8 layers of enrolled speech + c[:,1..j−1]
previously predicted codebooks
Target embeddings = sum of acoustic embeddings for layers 1 through j−1.
Prompt embeddings = sum of all 8 acoustic embeddings from enrolled speech.
STAGE INJECTION via Adaptive Layer Norm (AdaLN):
AdaLN(h, j) = a_j · LayerNorm(h) + b_j
where a_j, b_j = Linear(stage_embedding(j))
This injects the current stage number j into every layer without changing the architecture — one shared model handles all 7 stages.

🔗 Weight tying: W_output[j] = W_acoustic_embed[j+1]. The output projection for stage j shares weights with the input embedding of stage j+1.
Training: Each step, randomly sample stage j ∈ [2..8]. Predict all T tokens of codebook j simultaneously. Loss = CrossEntropy over all T positions.
OUTPUT: c[:,2..8] — 7 codebooks, each length T, predicted greedily (argmax)

Emergent Capabilities
Beyond simple cloning, VALL-E exhibits:
  • 🎲
    Diversity: Same text + same speaker → different outputs each run (different pace, emphasis, rhythm) due to sampling. Useful for data augmentation in ASR training.
  • 🏔
    Acoustic environment preservation: If the enrolled speech has reverb or background noise, synthesized speech matches — the model learned this from large-scale data, not explicit supervision.
  • 😠
    Emotion transfer: Angry/happy/sad enrolled speech → synthesized speech preserves that emotion, even though no emotion labels were used in training.
Implementation Checklist
What you need to build VALL-E
  • 1.
    EnCodec (off-the-shelf, frozen) — pip install encodec, use facebook/encodec_24khz. Do not train this, just use it as a tokenizer.
  • 2.
    G2P model — pip install phonemizer with espeak-ng backend. Converts text to phoneme sequences.
  • 3.
    AR Transformer — Standard decoder-only GPT architecture. Add phoneme embedding + acoustic embedding. Tie output projection weights to acoustic embedding. Causal attention mask. Output size = 1024 (codebook entries).
  • 4.
    NAR Transformer — Same arch but: (a) 8 separate acoustic embedding layers, (b) no causal mask (bidirectional), (c) replace LayerNorm with AdaLN(h, j) in every layer, (d) tie W_output[j] = W_acoustic_embed[j+1].
  • 5.
    AR training loop — Feed [phonemes | EOS | prompt_cb1 | EOS | target_cb1], compute causal cross-entropy on target_cb1 positions only.
  • 6.
    NAR training loop — Each step: sample j ∈ [2..8], feed [phonemes | full_C_tilde | c[:,1..j−1]], predict all T tokens of c[:,j] at once.
  • 7.
    Inference — AR with temperature sampling → c[:,1]; NAR greedy (argmax) 7× → c[:,2..8]; stack → C[T×8]; encodec.decode(C) → waveform.
🚫 Never use beam search in the AR model. It reliably causes the model to loop and repeat tokens indefinitely. Always use temperature sampling.
⚠️ Known limitations: Some words may be skipped, duplicated, or garbled (AR attention misalignment). Model struggles with accented speakers not in training data. Output diversity is good for ASR data augmentation but can be undesirable for production TTS.
